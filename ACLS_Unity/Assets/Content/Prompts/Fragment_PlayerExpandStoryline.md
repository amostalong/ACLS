## 当前任务：角色丰富化 + 故事线生成

基于已有的世界观设定、宏观势力格局、区域背景和近域网络数据，你的任务是：
1. **丰富主角设定**——生成更深入的角色背景、价值观、目标、秘密、人脉、已知情报和随身物品
2. **生成 1-2 条故事线**——这些故事线将作为后续 L1（当前场景）构建的叙事根基

## 输出要求（严格）

只输出一个 JSON 对象，包含以下顶层字段（尽量先输出 th 字段）：

```json

  "th": "你的推理过程，尽量先输出",

  "nar": "叙事文本，以 DM 口吻直接以「你」叙述。目标是让玩家想继续玩下去——每段叙述都应有足够的吸引力、悬念或张力",

  "pe": {
    "bg": "2-4 句角色背景叙事，应与此前已确定的角色设定一致",
    "val": "角色的核心价值观，1 句",
    "cg": "角色当前最想达成的目标，1 句",
    "sec": "角色的一个秘密或软肋，1 句",
    "conn": ["人脉1", "人脉2"],
    "kf": ["角色知道的情报1", "情报2"],
    "oi": ["随身物品1", "物品2"]
  },

  "sl": [
    {
      "ti": "故事线标题，10 字以内",
      "su": "故事线概述，1-2 句，交代核心冲突与走向",
      "inp": ["涉及的主要NPC名字"],
      "ini": ["涉及的关键物品"],
      "inl": ["涉及的重要地点"],
      "ktp": "关键时间点（如184年3月，或「三日后的月圆之夜」）",
      "hk": "当前可直接切入故事线的具体切入点/机会"
    }
  ],

  "ne": [
    {
      "n": "NPC 姓名",
      "bg": "该 NPC 的背景叙事，1-2 句",
      "val": "该 NPC 的核心价值观，1 句",
      "cg": "该 NPC 当前最想达成的目标，1 句",
      "sec": "该 NPC 的一个秘密或软肋，1 句"
    }
  ]
}
```

不要 JSON 之外的任何文字（包括 ``` 围栏）。

## 规则

- 所有内容必须与已有上下文一致，不可编造与已有设定矛盾的内容
- player_expansion（pe）的 conn/kf/oi 应结合 L2 数据（chars/fac/pl）生成，体现角色在当前环境中的真实位置
- sl 每条的 inp 必须来自已有的 L2 chars 或宏观/区域势力中的人物
- sl 的 ini / inl 需结合上下文推导，确保与世界观一致
- sl 的 ktp 应给出具体时间（如「184年3月」「三日后」），便于后续 L1 场景编排时间线
- sl 的 hk 要具体，能直接用于引导 L1 的初始场景设计
- 1-2 条故事线，每条要有不同的冲突方向
- **ne 必须覆盖主角附近的全部角色**，包括：
  - L2 chars 中所有角色（无论关系亲疏）
  - L3/L4 上下文中主角当前所在位置附近的可接触人物（即使主角尚未认识他们）
  - 每个角色都要展开 bg、val、cg、sec，越具体越好
- ne 的内容需与已有的 L2 数据和 pe 的内容一致，不可凭空编造
