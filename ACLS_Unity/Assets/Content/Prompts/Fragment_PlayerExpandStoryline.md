## 当前任务：角色丰富化 + 故事线 + 初始场景

基于已有的世界观设定、宏观势力格局、区域背景和近域网络数据，你的任务是一次性完成：
1. **丰富主角设定**——生成更深入的角色背景、价值观、目标、秘密、人脉、已知情报和随身物品
2. **生成 1-2 条故事线**——作为初始场景与后续剧情的叙事根基
3. **直接构建初始场景（L1）**——主角登场的第一幕：所在地点、场景描写、在场人物、即时处境、可去之处

## 规则

- 所有内容必须与已有上下文一致，不可编造与已有设定矛盾的内容
- player_expansion（pe）的 conn/kf/oi 应结合 L2 数据（chars/fac/pl）生成，体现角色在当前环境中的真实位置
- sl 每条的 inp 必须来自已有的 L2 chars 或宏观/区域势力中的人物
- sl 的 ini / inl 需结合上下文推导，确保与世界观一致
- sl 的 ktp 应给出具体时间（如「184年3月」「三日后」），便于后续 L1 场景编排时间线
- sl 的 hk 要具体，能直接用于引导 L1 的初始场景设计
- 1-2 条故事线，每条要有不同的冲突方向

### ⛔ L1 构建硬底线（不可省略）

构建 l1s 时必须遵守以下两条硬底线：

**硬底线①：聚落内 NPC 全部纳入**
- `an`（在场 NPC）必须覆盖主角当前位置同一聚落内的全部活跃 NPC
- 同一聚落 = 步行一炷香内可达的所有区域（城内街道、宅院、县衙、市集等）
- 不可切割、不可省略、不可只写"重要"的几个

**硬底线②：逻辑关联 NPC 全部纳入（不论距离）**
- `lrn`（逻辑关联 NPC）必须覆盖以下四类，不论距离远近：
  - **在途**：正在执行玩家交办任务的 NPC
  - **等待**：在等玩家决定、归来或下一指令的 NPC
  - **刚互动**：本叙事弧内与玩家有重要互动的 NPC
  - **热点关联**：与 🔴🟠 故事线核心事件相关的 NPC
- 每条 rt（关联原因）必须明确，便于后续重建时识别
- 数量上限 8-15 人；超过时优先保留"在途""等待"两类

### ⛔ L1 必须包含的四维信息

l1s 的每个非空字段都对应一个 LLM 在叙事时需要的认知维度：

| 字段 | 对应维度 | 用途 |
|------|---------|------|
| `cli` | 气候 | 当下场景的环境感，影响叙事感官细节 |
| `tod` | 时辰 | 决定 NPC 是否在场、衙署是否开门、城门是否关闭 |
| `an` | 物理近 NPC | 主对话可直接调度的角色 |
| `lrn` | 逻辑近 NPC | 虽不在场但需在叙事中呼应（信件、传闻、回忆） |
| `sln` | 故事线 | 玩家可介入的持续叙事线程，附下一拍方向 |
| `is` | 即时处境 | 当前场景的紧迫抓手 |
| `ex` | 出口 | 玩家可选择的方向，按紧迫度排序 |

`ex` 的紧迫度排序逻辑：🔴high = 时间锁临近或重大事件将至；🟡medium = 当日可用；🟢low = 长期可用。

### 故事线状态标记

- 🔴 **危急**：时间锁 < 3 日 或 直接威胁玩家
- 🟠 **紧张**：时间锁 3-7 日 或 重大压力
- 🟡 **进行中**：有明确下一拍，玩家可随时介入
- 🟢 **平稳**：背景运转，无紧迫压力

每条 sl 必须填 `st/cs/nb` 三字段；`sln` 是 sl 的子集投影，标题必须与 sl 中某条对应。

## 其他规则
- **ne 必须覆盖主角附近的全部角色**，包括：
  - L2 chars 中所有角色（无论关系亲疏）
  - L3/L4 上下文中主角当前所在位置附近的可接触人物（即使主角尚未认识他们）
  - 每个角色都要展开 bg、val、cg、sec，越具体越好
- ne 的内容需与已有的 L2 数据和 pe 的内容一致，不可凭空编造
- **l1s（初始场景）是主角登场的第一幕**，必须与上面生成的 pe / sl / ne 完全一致
  - l1s.an 的在场角色必须取自上面的 ne 或已有 L2 chars，不可新造人物
  - l1s.lrn 的逻辑关联 NPC 必须覆盖 ne + L2 中尚未纳入 an 的角色
  - l1s.sln 必须包含 sl 中所有故事线的状态投影
  - l1s 要给主角一个明确的即时处境（is）和 2-4 个出口（ex），让第一幕有抓手、能直接切入某条故事线
  - nar 即这一幕的开场旁白：此时主角已登场，使用第二人称「你」
