## 当前任务：L1 增量重建（StageRefresh）

玩家移动了位置、时间跨日、或场景内 NPC 大幅变动。当前的 L1 场景已过期，需要在已有 L2/L3/L4 上下文和上一版 L1 的基础上，**只重建 L1**，不要重写 L2/L3/L4，也不要重写主角或 NPC 的人设。

### 与初始 L1Build 的差异

| 维度 | StageCreate（首次） | StageRefresh（增量） |
|------|-------------------|---------------------|
| L2/L3/L4 | 一并构建 | **直接复用** `read_world_layer` |
| 主角人设 | 可重新生成 | **禁止改动** |
| NPC 人设 | 可重新生成 | **禁止改动** |
| L1 内容 | 全量构建 | 只改变化的字段（时辰/气候/NPC/出口） |
| 故事线 | 全量构建 | **保留**已有故事线，仅更新状态 |

### 工具使用（必须）

你需要调用以下工具核实当前数据，不可凭空编造：

1. `read_player_state` — 玩家当前位置、状态
2. `read_world_layer("L2")` — 当前近域网络
3. `read_world_layer("L3")` — 区域背景（仅当涉及跨郡判断时）
4. `lookup_character` — 任何出现在 an/lrn 的人物（必须已在 GameMemory 中）
5. `lookup_location` — 任何出现在 loc/ex 的地点

### 输出要求

只输出一个 JSON 对象，包含以下顶层字段（先输出 th）：

```json
{
  "th": "你的推理过程：本次刷新因（玩家移动/跨日/Δnpc），主要修改（cli/tod/an/lrn/ex）字段",

  "nar": "2-3 句过渡叙事文本（DM 口吻，第二人称「你」），承接上一幕末尾，描述主角进入新场景或场景刷新",

  "l1s": {
    "loc": "当前地点全称（必须与 read_player_state 一致）",

    "cli": {
      "w": "当前天气",
      "l": "当前光线",
      "wd": "风向与温度感受",
      "tmp": "温度档（寒/凉/爽/暖/热）"
    },

    "tod": {
      "sh": "十六时名",
      "cn": "俗称",
      "ls": "光线状态",
      "rm": "此时提示"
    },

    "an": [
      {"n": "角色名", "r": "关系/身份", "rv": 0, "st": "姿态/态度"}
    ],

    "lrn": [
      {"n": "逻辑关联 NPC", "loc": "所在", "rt": "在途|等待|刚互动|热点关联", "rv": 0, "ca": "在做什么"}
    ],

    "sln": [
      {"ti": "故事线标题", "st": "🔴|🟠|🟡|🟢", "cs": "当前状态", "nb": "下一拍", "inp": ["NPC"]}
    ],

    "is": "主角即时处境",
    "ex": [
      {"d": "目的地", "u": "high|medium|low", "n": "说明"}
    ]
  }
}
```

### 硬底线（继承自初始 L1Build）

- **聚落内 NPC 全部纳入**（an 覆盖步行一炷香内可达的所有活跃 NPC）
- **逻辑关联 NPC 全部纳入**（lrn 覆盖在途/等待/刚互动/热点四类）
- **故事线状态投影**（sln 包含所有 sl 中的故事线，标题必须一致）
- **exit 紧迫度排序**（🔴high 必出，🟡medium ≥ 1，🟢low 可选）

### 增量 vs 全量判定

| 触发 | 主要修改字段 | 保留字段 |
|------|------------|---------|
| 仅跨时辰 | cli, tod | loc, an, lrn, sln, is, ex |
| 仅 Δnpc 变化 | an | loc, cli, tod, lrn, sln, is, ex |
| 玩家移动到新聚落 | loc, sd, cli, tod, an, ex, sln(过滤掉失去关联的) | lrn(更新), is(重写) |
| 跨日 | cli, tod, sln(状态推进), an(更新作息), ex | loc, lrn(若 NPC 仍在远处) |

### 重要规则

- 所有 NPC 必须用 `lookup_character` 验证其存在
- 所有地点必须用 `lookup_location` 验证其存在
- 不要 JSON 之外的任何文字（包括 ``` 围栏）
- nar 必须承接上一幕——本次刷新是「场景自然过渡」，不是「重置」
