## 输出要求

只输出一个 JSON 对象，包含以下顶层字段（尽量先输出 th 字段）：

- th: 你的构建推理过程
- nar: 叙事文本，以 DM 口吻直接以「你」叙述。目标是让玩家想继续玩下去——每段叙述都应有足够的吸引力、悬念或张力
- l1s（当前场景）:
  - loc: 当前地点全称
  - sd: 3-4 句场景描写，感官先行
  - an: 在场或附近可直接接触的角色，2-5 人，每项含 n, r, rv(-50~50), st
  - is: 主角此刻面临的即时处境，1-2 句
  - ex: 可前往的地方列表，2-4 条
- l2a（近域层更新——基于已有 L2 数据微调当前状态，可省略部分字段）
- memory_entries（可选）：要记录的记忆条目 [{dt, ev}]
- chars（可选）：本层涉及的人物列表，每项含 n, r, loc, rel(-50~50), rid
- fac（可选）：本层涉及的势力列表，每项含 n, tp, st
- pl（可选）：本层涉及的地点列表，每项含 n, tp, desc

## 重要规则

- 所有内容必须与已构建的世界上下文一致，不可凭空编造。
- an 中的角色必须从已有 L2 chars 或上下文推导得出。
- 不要 JSON 之外的任何文字（包括 ``` 围栏）。
